보조배터리 앱 구조 시각화

이 페이지는 학생 화면, 관리자 화면, 데이터 저장 방식, 그리고 주요 함수 흐름을 한눈에 볼 수 있게 정리한 문서입니다.

학생 화면

학생이 쓰는 흐름

  • 학번/이름 선택
  • 배터리 번호 입력
  • 대여 또는 반납 실행
  • 반납 예정 시간 표시
관리자 화면

관리자가 쓰는 흐름

  • 비밀번호 인증
  • 배터리 상태/통계 확인
  • 배터리 이름 변경 또는 삭제
  • 이력/보관 기록 확인
데이터 저장

저장소 구조

  • localStorage에 JSON 형태로 저장
  • batteries 배열과 logs 배열 관리
  • 로그는 90일 보관 후 정리됨

데이터 모델

Battery
  • id: 배터리 번호
  • name: 표시 이름
  • status: available / rented
  • borrower, borrowedAt, dueAt
Log
  • at: 발생 시각
  • action: 이벤트 종류
  • detail: 세부 내용
Stats
  • total: 전체 배터리 수
  • available: 대여 가능 수
  • rented: 대여 중 수
  • overdue: 연체 수

확인 포인트

이 앱은 정적 페이지 기반이라 브라우저에서 바로 열 수 있으며, 상태는 모두 로컬 저장소에 남습니다. 데이터가 꼬였을 때는 브라우저의 로컬 스토리지를 비우면 초기화할 수 있습니다.
